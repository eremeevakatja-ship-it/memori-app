# Керівник — Memori App

> Папка керівника проекту. Читати на початку кожної сесії по memoriapp.

---

## Початок сесії (обов'язково)

1. Прочитай `_manager/STATUS.md` — що зараз в роботі
2. Прочитай `_manager/BACKLOG.md` — поточний стан фіч
3. Прийми завдання від користувача → делегуй агентам

---

## Команда агентів

| Агент | Папка | Коли задіяти |
|-------|-------|-------------|
| Product Manager | `AGENTS/product-manager/` | Нова фіча → уточнення вимог, User Story |
| UX Designer | `AGENTS/ux-designer/` | Фіча потребує нових екранів або flow |
| Methodologist | `AGENTS/methodologist/` | Зміна алгоритму навчання, нові режими |
| Frontend Dev | `AGENTS/frontend-dev/` | Реалізація в коді |
| QA Tester | `AGENTS/qa-tester/` | Перевірка після кожної реалізованої фічі |
| Backend Dev | `AGENTS/backend-dev/` | Тільки з Фази 4+ |

---

## Workflow для нової фічі

```
Користувач → Керівник
  → PM Agent (User Story + AC)
  → [UX Agent] (якщо новий UI)
  → [Methodologist] (якщо зміна алгоритму)
  → Frontend Dev (реалізація)
  → QA Tester (тестування)
  → Керівник → Користувач (звіт)
```

### 🔴 Тестування в браузері після зміни app.js/style.css

`index.html` підключає `app.js?v=N` і `style.css?v=N` з ручним номером версії. Є проксі-кешування (не тільки Service Worker), яке ігнорує `cache: 'no-store'` і навіть повне очищення CacheStorage — стара версія файлу віддається, поки не зміниться сам номер `?v=N` в URL. **Перед кожною перевіркою змін у браузері:**
1. Збільшити `?v=N` на +1 і для `app.js`, і для `style.css` в `index.html` (обидва незалежні лічильники — легко забути один)
2. Навігувати на URL з унікальним доданим query (напр. `?bust=<щось нове>`) — саме на HTML, бо навіть index.html кешується проксі за тим самим принципом
3. Перевірити `document.querySelector('script[src*=app]').src` і `link[rel=stylesheet].href` в консолі — переконатись що там новий `?v=`

Без цього можна проганяти тести на застарілому коді й не помітити (симптом: функція/змінна не визначена, або стилі не застосувались, хоча в файлі на диску все правильно).
