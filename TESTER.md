## Тест-сценарії

### Сценарій 1 — Перший візит (Мама Оля)
1. Відкрила застосунок холодно (порожнє поле)
2. Чи зрозуміло що вставляти? Чи є підказка?
3. Чи не збиває кількість налаштувань?
4. Вставляє вірш (4 рядки, ~30 слів)
5. Натискає "Почати" без будь-яких змін налаштувань
6. Проходить перший блок — чи зрозумілі інструкції?

**Вірш для тесту:**
```
Котилася торба з великого горба,
А в тій торбі хліб-паляниця.
Хто любить цю казку — той буде щасливий,
Хто не любить — тому не журиться.
```

### Сценарій 2 — Довгий текст (Тато Андрій)
1. Вставляє 3 абзаци (~250 слів)
2. Обирає "Близько до тексту"
3. Перевіряє чи правильно текст поділено на блоки (не рвуться речення?)
4. Проходить 2-3 блоки, перевіряє метод "Письмо"

**Текст для тесту:**
```
Сучасний менеджмент вимагає від керівника не лише технічних знань, але й розвинених навичок комунікації. Здатність чітко формулювати думки, слухати співрозмовника та знаходити компроміс — це основа ефективного лідерства.

Дослідження показують, що команди з відкритою культурою зворотного зв'язку досягають результатів на 25% краще. Це означає, що інвестиції у розвиток м'яких навичок окупаються набагато швидше, ніж технічні тренінги.

Тому кожен керівник повинен щодня практикувати активне слухання, задавати відкриті питання та заохочувати ініціативу в команді. Тільки так можна побудувати організацію, здатну адаптуватись до змін.
```

### Сценарій 3 — Короткий текст з помилкою (Богдан)
1. Вставляє 1 речення (3 слова) — має показати помилку
2. Вставляє порожнє поле — має показати помилку
3. Натискає "Почати" кілька разів підряд швидко
4. Під час паузи — міняє тривалість

### Сценарій 4 — Перерваний сеанс (Катя)
1. Вставляє промову (~150 слів), обирає "Близько до тексту"
2. Проходить 2 блоки → закриває вкладку (симуляція дзвінка по роботі)
3. Відкриває знову → чи показується банер "Продовжити"?
4. Натискає продовжити → чи починається з правильного блоку?

**Текст для тесту:**
```
Шановні колеги, дозвольте привітати вас з нашим корпоративним святом. Цей рік був насичений викликами, але ми впоралися разом — завдяки кожному з вас.
Ми запустили три нові продукти, розширили команду вдвічі і вийшли на три нові ринки. Це результат вашої роботи, вашої відданості та вашого таланту.
Тому сьогодні я хочу просто сказати: дякую. Дякую за довіру, за ідеї, за той запал, з яким ви приходите щоранку. Попереду нас чекає ще більше — і я впевнена, що ми справимося.
```

### Сценарій 5 — Дитина сама (Діма)
1. Мама запустила додаток, вставила вірш і пішла
2. Діма бачить перший екран — чи є велика зрозуміла кнопка?

5. Натискає "Готово" до кінця таймера — чи пускає?

### Сценарій 6 — OCR (будь-хто)

## Що перевіряти в коді (без браузера)

### Логіка розбиття тексту
- `smartSplitText()` — чи не рве речення посередині?
- Мінімум 4 слова на блок — перевір з тестовими текстами

### Валідація
- Порожній текст → `validation_no_text`
- Менше 4 слів → `validation_too_short`
- Чи скидається помилка при наступному вводі?

### Стан сесії
- `saveState()` / `loadState()` — чи правильно відновлюється?
- `checkSavedState()` — чи показується банер?

### Статистика
- `updateStats()` — чи правильна логіка streak?
- Якщо 0 блоків — чи не оновлює?

### Переклади
- Чи всі ключі присутні у всіх 6 мовах?
- Знайди відсутні ключі
